Fraud_Detection_Graph

Predictive Financial Fraud Detection Using Behavioral Graph Learning

12-slide elaborative deck based on your notebook pipeline + improvements inspired by the shared GNN repo and production architecture patterns.

Slide 2 - Problem Context

  • Fraud is networked behavior, not isolated events.
  • Rule-only systems miss coordinated rings and new attack patterns.
  • Class imbalance makes naive models overpredict legitimate transactions.

Slide 3 - Repo Research Takeaways

  • Shared repo emphasizes graph representation, imbalance, and privacy concerns.
  • GNN framing is suited for entity linkage: account-device-IP-merchant relationships.
  • Our implementation keeps this graph-first philosophy and adds production APIs.

Reference: github.com/bimbomuri/Fraud_Detection_GNN (README + notebook structure)

Slide 4 - End-to-End Architecture

  • Frontend dashboard: training, prediction, simulation, and diagnostics.
  • Backend FastAPI: REST endpoints for scoring, graph, ML metrics, and research.
  • Middle layer: data generation, graph construction, feature engineering, model inference.

Slide 5 - Data Model and Generation

  • Synthetic Elliptic-style transactions with sender, receiver, merchant, device, IP.
  • Fraud ring injection with shared devices/IPs and burst-window timing.
  • Behavior flags: night activity, high-value ratio, cross-border ratio.

Slide 6 - Graph and Feature Engineering

  • Directed account graph with edge transaction/fraud aggregates.
  • Node features: sent/received stats, degree, pagerank, clustering.
  • Extended features: device diversity, IP diversity, merchant diversity.

Slide 7 - Modeling Strategy

  • Baseline model: class-balanced RandomForest for robust tabular + graph features.
  • Diagnostic outputs: confusion matrix and top feature importances.
  • Design keeps interface compatible for future GraphSAGE drop-in upgrade.

Slide 8 - Evaluation and Diagnostics

ROC-AUC~0.77
F1~0.75
Recall~0.80
Precision~0.70
  • Metrics exposed through `/api/ml/metrics` and `/api/ml/report`.
  • Dashboard shows feature importances and confusion matrix.

Slide 9 - Real-Time Inference APIs

  • `/api/transactions/score` for transaction-level risk decisions.
  • `/api/ml/predict` for account-level fraud probability and risk tier.
  • Unknown account handling improved: deterministic median-profile fallback.

Slide 10 - Research and Data Intelligence Layer

  • `/api/ml/data-profile` gives fraud rate, channel/country mix, risky devices.
  • `/api/ml/research` publishes findings + recommendations for investigators.
  • Turns model output into actionable fraud operations guidance.

Slide 11 - Deployment and Product Readiness

  • FastAPI app with docs, route grouping, and strict request/response schemas.
  • Frontend UI for investigators and model operators on one URL.
  • Automated tests validate service and API behavior.

Slide 12 - Roadmap (Next Level)

  • Integrate PyTorch Geometric GraphSAGE model with checkpoint management.
  • Add drift monitoring and weekly threshold calibration by channel.
  • Enable analyst feedback loop to retrain model from confirmed cases.

Fraud_Detection_Graph is now a full-stack, research-backed, presentation-ready fraud graph platform.

Project Overview

  • End-to-end graph learning inspired fraud detection workflow.
  • Backend + middleend + frontend integrated in one running app.
  • Research, diagnostics, and model governance included.

Environment Setup

Dependencies are installed from requirements.txt; use this page to execute pipeline steps.

Imports

Core stack: FastAPI, pandas, numpy, networkx, scikit-learn, and browser-side JS for notebook interaction.

Step 2: Graph Construction

Graph edges/nodes are created in training pipeline and reflected in report/metrics outputs.

Step 3: Feature Engineering

Node-level behavioral features are computed (volume, timing, diversity, graph centrality).

Step 4: Graph to Data Object

Graph features are transformed into model-ready tabular vectors for account-level classification.

Step 5: Model Building - GraphSAGE Style Objective

Current production baseline uses a robust tree ensemble; interface is GNN-upgrade ready.
